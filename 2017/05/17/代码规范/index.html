#nil检查
使用 nil 的检查来检查应用程序的逻辑流程，而不是避免崩溃。Objective-C 运行时会处理向 nil对象发送消息的情况。如果方法没有返回值，就没关系。如果有返回值，可能由于运行时架构、返回值类型以及 OS X 版本的不同而不同。
集合类型 一定要判断是否为nil

注意，这和 C/C++ 中检查指针是否为 ‵‵NULL`` 很不一样，C/C++ 运行时不做任何检查，从而导致应用程序崩溃。因此你仍然需要保证你不会对一个 C/C++ 的空指针解引用。

#BOOL 若干陷阱
BOOL 会被编译器认为是8位二进制数，是一个带符号的字符类型（singed char），YES 定义成1，NO定义成NO，在赋值的时候要注意，不要直接把整形转换成BOOL，常见的错误包括将数组的大小、指针值及位运算的结果直接转换成 BOOL ，取决于整型结果的最后一个字节，很可能会产生一个 NO 值。当转换整形至 BOOL 时，使用三目操作符来返回 YES 或者 NO。
bool 0 是false，非0 是true。

文件名

文件名须反映出其实现了什么类 – 包括大小写

方法名

  1. 如果方法的返回值是新创建的，那么方法名的首个词应是返回值的类型。
  2. 把表示参数类型的名词放在参数前面。
  3. 如果方法要在当前对象上执行擦破做，那么就应该包含动词。
  4. 不要使用str这种简称，应该用string这样的全称。
  5. Bool属性应加is前缀，如果方法返回非属性的Bool值，应该根据其功能，选用has或is当前缀。

类与协议命名

应该为类与协议的名称加上以三个大写字母作为前缀，以避免命名空间冲突，而且应该像给方法起名那样把词句组织好，十七从左至右读起来通顺。当创建一个子类的时候，你应该把说明性的部分放在前缀和父类名的中间
比如一个UIViewController的子类：ZOCTimelineViewController。
鱼泡泡项目类名前加Ypp

常量

常量名（如宏定义、枚举、静态局部变量等）应该以小写字母 k 开头，使用驼峰格式分隔单词，如：kInvalidHandle，kWritePerm。

字面值

尽量使用字面值来创建不可变的NSString，NSDictionary，NSArray和NSNumber对象。不要把nil传进NSArray和NSDictionary。

#注意！

  1. 写代码需添加必要的注释，以百分之五十为标准
  2. load函数不要写代码，如必要需商量并注明
  3. PCH文件不允许加任何东西

#预防crash事项

  1. 避免在网络回调里面写过多业务逻辑代码，建议使用weakself调用函数处理
  2. cellForRowAtIndexPath中不允许出现返回nil的情况
  3. 界面退出后网络请求须停止
